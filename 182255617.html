HACS200-Confluence : MITM issues we have run into + Debugging

Created by Kaitlyn Yang on Nov 25, 2020

issue

solution

  • tail MITM log - failed auth

  • SSH shows MITM prompt

  • Login Attempts show attackers are consistently attempting

  • If one machine is working and not the others, diff the sshd_config files to see if there is a line commented out

  • If they are the same, check networking rules: ip addr, ip route, etc. Make sure the netmasks and gateways are correct

  • Login Attempts show attackers are consistently attempting

  • SSH shows wrong MITM prompt BUT not the default one

  • Check with TAs that your IP address is not linked to another groups

  • SSH shows wrong MITM prompt and it is the standard one

  • Make sure that MITM is running, try switching the port if necessary

  • SSH hangs, does not give MITM prompt

  • telnet 172.20.0.1 <port> hangs, does not say SSH but also does not say connection refused

  • Check iptable rules iptables -L and look at the chain INPUT to make sure that packets to the IP address are not being dropped

general debugging options:

  • recycle container, sometimes MITM fixes itself

  • clear IP table rules and restart
